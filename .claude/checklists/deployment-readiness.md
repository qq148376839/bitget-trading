# 部署就绪检查清单

在部署到生产环境前，必须完成以下所有检查项。

## 📋 使用说明

- 🔴 = 阻断项（必须完成，否则不能部署）
- 🟡 = 推荐项（强烈建议完成）
- 🟢 = 可选项（可择机完成）

## 1. 代码质量检查 🔴

### 1.1 类型安全
- [ ] 所有 TypeScript 文件没有 `any` 类型
- [ ] 所有函数定义了参数和返回类型
- [ ] `pnpm run type-check` 通过
- [ ] ESLint 检查通过（`pnpm run lint`）

### 1.2 错误处理
- [ ] 所有 async 函数有 try-catch
- [ ] 所有外部 API 调用有错误处理
- [ ] 使用统一的 AppError 类型
- [ ] 错误日志包含足够上下文信息

### 1.3 日志完整性
- [ ] 关键操作有日志记录（下单、撤单、资金变动）
- [ ] 日志级别合理（info/warn/error）
- [ ] 没有敏感信息泄露（SecretKey/Passphrase）
- [ ] 使用 LogService 而非 console.log

## 2. 交易安全检查 🔴

### 2.1 资金安全
- [ ] 下单前检查资金充足性（含手续费+滑点）
- [ ] 资金操作使用数据库事务
- [ ] 资金计算使用字符串运算（避免浮点误差）
- [ ] 资金变动有审计日志
- [ ] 本地余额定期与 Bitget 对账

### 2.2 订单安全
- [ ] 订单参数严格验证（symbol/side/orderType/force）
- [ ] tradeSide 参数按持仓模式正确设置（合约）
- [ ] 订单精度符合交易对规则（pricePrecision/quantityPrecision）
- [ ] 订单状态与 Bitget 同步
- [ ] post_only 订单有自适应逻辑

### 2.3 策略安全
- [ ] 止损参数已设置（建议 2%-5%）
- [ ] 最大回撤已设置（建议 10%-20%）
- [ ] 日亏损限制已设置
- [ ] 单笔订单金额有上限
- [ ] 总仓位有上限（建议 ≤ 50%）

### 2.4 风险控制
- [ ] 策略参数在合理范围（网格数量、价差、利润目标）
- [ ] 网格价差 > 手续费*2
- [ ] 剥头皮利润目标 > 手续费*2
- [ ] 单网格金额 > 最小下单量
- [ ] 价格区间包含当前价格

## 3. 测试覆盖检查 🔴

### 3.1 单元测试
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] 所有关键路径有测试（资金、订单、策略）
- [ ] 边界条件有测试（余额刚好、精度边界）
- [ ] 异常场景有测试（API 失败、网络超时）
- [ ] 所有测试通过（`pnpm test`）

### 3.2 集成测试
- [ ] 模拟盘测试至少 1 小时无异常
- [ ] 订单能正常提交和撤销
- [ ] 订单状态正确同步
- [ ] 资金变动正确记录
- [ ] 策略能正常启动和停止

### 3.3 端到端测试 🟡
- [ ] 前端能正常展示策略状态
- [ ] 前端能正常创建策略
- [ ] 前端能正常停止策略
- [ ] 实时数据能正常刷新
- [ ] 错误提示能正常显示

### 3.4 压力测试 🟡
- [ ] 模拟盘运行 24 小时无异常
- [ ] 高频交易场景测试（100+ 订单/小时）
- [ ] 并发资金操作测试
- [ ] API 限频测试（确认重试机制）

## 4. 安全审计检查 🔴

### 4.1 敏感信息管理
- [ ] APIKey 存储在环境变量中
- [ ] SecretKey 存储在环境变量中
- [ ] Passphrase 存储在环境变量中
- [ ] .env 文件在 .gitignore 中

### 4.2 注入防御
- [ ] 所有数据库查询使用参数化

- [ ] 没有拼接外部输入到系统命令

### 4.3 API 安全
- [ ] HMAC-SHA256 签名正确实现
- [ ] 时间戳防重放攻击（合理窗口期）
- [ ] 请求头正确设置（ACCESS-KEY/SIGN/TIMESTAMP/PASSPHRASE）
- [ ] 模拟盘/实盘通过 paptrading 请求头区分
- [ ] Bitget IP 白名单已配置（生产环境）

### 4.4 依赖安全 🟡
- [ ] 运行 `pnpm audit` 无高危漏洞

- [ ] 没有引入不必要的依赖

## ⚠️ 重要提醒

1. **资金安全优先** — 任何阻断项未完成，禁止部署到生产环境
2. **小步快跑** — 建议先模拟盘验证，再小额资金验证，最后全量部署
3. **持续监控** — 部署后至少监控 1 小时，观察关键指标
4. **准备回滚** — 遇到异常立即回滚，不要侥幸
5. **记录变更** — 部署后更新 claude-progress.txt 和 features.json

## 🚀 部署命令速查

```bash
# 1. 检查环境变量
cat .env | grep -E "BITGET_|DATABASE_URL"

# 2. 运行数据库迁移
cd api && pnpm run migrate:up && cd ..

# 3. 构建镜像
docker-compose build

# 4. 启动服务
docker-compose up -d

# 5. 查看日志
docker-compose logs -f

# 6. 健康检查
curl http://localhost/api/health

# 7. 回滚（如需要）
git revert HEAD
docker-compose down
docker-compose up -d
```
