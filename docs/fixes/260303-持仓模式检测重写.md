# 持仓模式检测重写 — 统一 API 查询 + 持仓列表推断 + 卖单前验证

**日期**: 2026-03-03
**类型**: Bug Fix
**影响范围**: 合约交易（剥头皮 + 网格策略）

---

## 问题背景

持仓模式检测经历了多轮迭代（7 次提交），但核心方法 `getHoldMode()` 仍存在三个根本性错误，导致模拟盘/实盘合约交易时频繁出现 22002（暂无仓位可平）和 40774（持仓模式不匹配）错误。

### 错误 1：错误假设模拟盘不支持 position-mode GET API

旧代码在 `catch` 分支中检查 `simulated` 标志，认为模拟盘的 `GET /api/v2/mix/account/position-mode` 会返回 404。实际上模拟盘通过 `paptrading: 1` 请求头使用完全相同的 API，不需要特殊处理。

### 错误 2：detectSimulatedHoldMode 参数缺失

`detectSimulatedHoldMode()` 调用 `set-position-mode` API 时只传了 `{ productType, posMode }`，缺少必需的 `symbol` 和 `marginCoin` 参数，导致 API 必定失败，进入重试循环后最终返回错误的默认值。

### 错误 3：未利用持仓列表推断

用户反馈"方向是在持仓列表里看的吧"—— `GET /api/v2/mix/position/all-position` API 返回的 `holdSide` 和 `posMode` 字段可以直接推断持仓模式，但旧代码未使用此信息。

---

## 修复方案

### 新的 getHoldMode() 检测流程

```
1. UTA 账户 → 直接返回 single_hold
2. GET /api/v2/mix/account/position-mode → 解析 posMode（实盘 + 模拟盘通用）
3. 若步骤 2 失败 → GET /api/v2/mix/position/all-position → 从 posMode/holdSide 推断
4. 所有方法失败 → 默认 double_hold（安全策略，确保 tradeSide 始终发送）
```

### 持仓列表推断逻辑

| holdSide 值 | 推断结果 | 说明 |
|-------------|---------|------|
| `net` | `single_hold` | 单向持仓模式下，持仓方向为 net |
| `long` / `short` | `double_hold` | 双向持仓模式下，多空分开展示 |

### 卖单前持仓验证（新增）

在卖单下单前查询持仓列表，实现两个目标：
1. **确认仓位已到位** — 若持仓未到位，额外等待 3 秒
2. **验证/纠正 holdMode** — 从持仓的 `holdSide` 字段实时校正持仓模式

```typescript
export interface FuturesPosition {
  marginCoin: string;
  symbol: string;
  holdSide: 'long' | 'short' | 'net';
  posMode: string;
  total: string;
  // ... 其他字段
}
```

### 2. `api/src/services/futures-account.service.ts`

| 操作 | 内容 |
|------|------|
| **删除** | `import { getBitgetConfig }` — 不再需要区分模拟盘/实盘 |
| **删除** | `detectSimulatedHoldMode()` — 基于错误假设且参数缺失 |
| **新增** | `getPositions(productType, marginCoin)` — 查询所有持仓（公开方法） |
| **新增** | `queryPositionModeAPI(productType)` — 封装 position-mode API |
| **新增** | `inferHoldModeFromPositions(productType)` — 从持仓列表推断模式 |
| **重写** | `getHoldMode()` — 四层递进检测策略 |

### 3. `api/src/strategy/scalping-strategy.engine.ts`

在 `handleBuyFilled()` 中，`sleep(5000)` 之后、重试循环之前新增持仓检查：

```typescript
// 卖单前检查持仓是否已到位
const positions = await futuresAccountSvc.getPositions(productType, marginCoin);
const pos = positions.find(p => p.symbol === symbol && parseFloat(p.total) > 0);
if (!pos) {
  // 持仓未到位，额外等待
  await this.sleep(3000);
} else {
  // 从持仓 holdSide 验证/纠正 holdMode
  const inferredMode = pos.holdSide === 'net' ? 'single_hold' : 'double_hold';
  if (inferredMode !== this.holdMode) {
    this.holdMode = inferredMode;
    this.mergeEngine?.updateHoldMode(inferredMode);
  }
}
```

### 4. `api/src/strategy/grid-strategy.engine.ts`

在 `placeSellForLevel()` 中，`sleep(800)` 之后新增同样的持仓检查逻辑（无 mergeEngine 更新，网格策略不使用合并引擎）。
